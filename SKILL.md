# Teambition项目状态查看技能

## 概述

此技能用于查看和分析Teambition项目状态，包括任务统计、进度跟踪、团队工作负载分析等。

## 前置条件

1. 浏览器控制服务必须可用
2. 用户需要已经在浏览器中打开Teambition项目页面
3. 确保已登录Teambition账户

## 核心工作流程

### 1. 检查浏览器状态

首先检查浏览器控制服务是否可用：

```bash
browser action=status
```

如果浏览器未运行，启动浏览器：

```bash
browser action=start profile=openclaw
```

如果遇到连接问题，指导用户：
- 如果使用Chrome扩展模式，确保已点击OpenClaw扩展图标连接标签页
- 如果使用openclaw模式，确保OpenClaw gateway正在运行

### 2. 获取打开的标签页

列出当前打开的浏览器标签页：

```bash
browser action=tabs profile=openclaw
```

识别Teambition项目标签页（通常包含"teambition.com"和项目名称）

### 3. 捕获页面快照

使用正确的targetId捕获Teambition页面快照：

```bash
browser action=snapshot targetId=<tab-target-id> profile=openclaw
```

### 4. 分析页面内容

从快照中提取关键信息：

#### 项目基本信息
- 项目名称
- 当前视图名称
- 需求总数统计

#### 需求分类统计
- 各个需求分类的任务数量
- 重点关注"待处理"、"开发中"、"已完成"等状态

#### 看板视图分析
- 每个看板列的任务数量
- 任务标题和状态
- 截止日期和计划工时
- 创建者信息

#### 时间节点分析
- 今天截止的任务
- 近期重要截止日期
- 已逾期的任务

#### 团队参与情况
- 主要创建者列表
- 各成员的任务分配情况

### 5. 生成状态报告

按照以下结构生成报告：

```
## 📊 Teambition项目状态概览

### 项目基本信息
- 项目名称
- 当前视图
- 需求总数

### 需求分类统计
- 列出各分类数量

### 看板视图状态
- 按状态分组（待处理、开发中等）
- 列出关键任务及其详情

### 关键时间节点
- 今天截止的任务
- 近期重要截止

### 团队参与情况
- 主要参与者
```

## 常见分析维度

### 1. 项目健康度评估
- 待处理任务数量
- 开发中任务数量
- 逾期任务数量
- 整体完成率

### 2. 工作负载分析
- 按创建者统计任务数量
- 识别工作负载过重的成员
- 分析任务分配均衡性

### 3. 风险识别
- 即将截止的任务（3天内）
- 长期未更新的任务
- 缺少计划工时的任务

### 4. 进度跟踪
- 里程碑完成情况
- 版本发布进度
- 关键路径任务状态

## 高级功能

### 切换不同视图

如果需要查看特定视图，可以：
1. 点击视图选择器
2. 选择目标视图
3. 重新捕获快照并分析

### 任务详情查看

对于特定任务：
1. 点击任务卡片
2. 捕获任务详情快照
3. 提取任务描述、子任务、评论等信息

### 历史数据对比

如果需要对比历史状态：
- 定期保存快照数据
- 比较不同时间点的状态变化
- 生成进度趋势报告

### 详细报告

## 注意事项

1. **页面加载**：确保页面完全加载后再捕获快照
2. **权限问题**：某些敏感信息可能需要特定权限
3. **数据更新**：Teambition数据可能实时更新，注意快照时间
4. **分页处理**：如果任务很多，可能需要滚动多次或分页处理
5. **网络稳定性**：确保网络连接稳定，避免数据不完整

## 故障排除

### 浏览器连接失败
- 检查OpenClaw gateway状态
- 重启浏览器控制服务
- 尝试使用不同的profile

### 页面加载超时
- 增加等待时间
- 检查网络连接
- 刷新页面重试

### 快照数据不完整
- 确认页面滚动到正确位置
- 重新捕获快照
- 检查页面结构是否变化
